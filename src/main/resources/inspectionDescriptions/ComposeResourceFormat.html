Reports invalid format specifiers in Compose Multiplatform string and plural resources.

Compose Multiplatform only supports positional format specifiers in the form %N$s (string) and %N$d (integer), where N is a 1-based index. Unlike Java's String.format(), unpositioned specifiers like %s or %d, as well as width/precision modifiers and other conversion types (%f, %x, %b), are not supported.

This inspection also verifies that specifier positions are sequential (starting from 1 with no gaps) and that all plural variants use a consistent set of format specifiers.

Example of problematic code:


<string name="greeting">Hello %s, you have %d items</string>

After the quick fix is applied:


<string name="greeting">Hello %1$s, you have %2$d items</string>

Use the provided 'Convert to positional format specifier' quick fix to automatically replace unpositioned specifiers with the correct positional syntax.
