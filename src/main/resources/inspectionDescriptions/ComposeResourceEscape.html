Reports unnecessary Android-style backslash escapes in Compose Multiplatform string and plural resources.

Unlike Android, Compose Multiplatform does not require escaping ', ", @, or ? with a backslash. Using these Android escapes in Compose resources cause the backslash to appear literally in the rendered string.

Example of problematic code:


<string name="greeting">Let\'s do something</string>

After the quick fix is applied:


<string name="greeting">Let's do something</string>

The following backslash escapes are flagged as unnecessary:

  • \' — apostrophe does not need escaping
  • \" — double quote does not need escaping
  • \@ — @ is not treated as a resource reference
  • \? — ? is not treated as a theme attribute

Valid backslash sequences (\n, \t, \\, \uXXXX) and standard XML entities (&amp;, &lt;, &apos;) are not affected by this inspection.
