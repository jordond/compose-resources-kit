Reports mismatches between the number of format arguments passed to stringResource() or pluralStringResource() and the number of format specifiers defined in the corresponding XML resource.

For example, if a string resource contains %1$s and %2$d, the call site must provide exactly 2 format arguments after the resource reference. For pluralStringResource(), note that the quantity parameter is not counted as a format argument.

Example of a stringResource call with a missing argument:


// strings.xml: <string name="greeting">Hello %1$s, you have %2$d items</string>

// Warning: expects 2 format arguments, but 1 was provided
stringResource(Res.string.greeting, name)

Correct usage:


stringResource(Res.string.greeting, name, count)

This inspection resolves stringResource and pluralStringResource calls from the org.jetbrains.compose.resources package and cross-references the format specifiers found in the XML resource definition.
